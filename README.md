一款轻量、无后端的 Obsidian 插件。由于小米未提供官方导出通道，本工具允许你直接在 Obsidian 内部安全隔离地绑定小米云同步，实现笔记向本地知识库的完整、无缝与极速增量迁移。

---

## ✨ 核心特性与极限优化

不仅实现了基础的笔记还原，本项目更在长期的打磨中针对跨平台环境的各类深度盲区做出了极致优化：

- **🚀 零外部依赖 & 原生安全沙盒登录**：无需部署额外后端或繁琐抓包。内置专属 Electron Webview 网络引擎，直连云端。完美规避 CORS、HttpOnly 等请求封锁。直接扫码验证，本地驻留凭证。
- **⚡️ 极速增量同步 & 并发护航**：智能识别 `syncTag` 游标拉取差异记录，极大地节省网络带宽。内置全局 `isSyncing` 竞态锁，有效阻击连击造成的写盘冲突。
- **📅 MiNote2Daily 日记集成**：支持将当日产生或修改的所有小米笔记一键（或自动）同步至 Obsidian 日记（Daily Note）。支持追加、标题定位及光标插入多种灵活模式。
- **📝 强力 Markdown 编译引擎**：精准转译小米私有排版标签（包含颜色、待办、对齐、分割线等），并自带文件名非法字符净化兜底，杜绝因标题异形（反斜杠/生僻符等）导致的落盘崩溃。
- **🛡️ 物理级隐私隔离 (Partition Rotation)**：创新性引入了「沙盒分区轮换」退登机制。当你希望安全退出时，插件直接将被污染的物理虚拟微盘永久切除销毁，为你原地签发一个全新净白的无纸空间，百分之百断绝“旧缓存死灰复燃”导致的静默越权登录。
- **📡 同源守卫与拦截防护 (`ensureOrigin`)**：针对同步中途可能因云端官方 SPA 前端页面（Vue.js）发生静默心跳续期、主动打断后台批量 `Fetch` 并导致报错的终极痛点，加入底层强行驻停钩子。在发包前物理泊车于云端纯净静态源（如 `favicon.ico`），彻底剥离云端 JS 轮询黑手，确保上千篇文稿拉取顺畅无虞。
- **双模附件处理**：图片音频既可集中「全量下载至本地」，亦可开启极致省空间的「在线跨域直链直插」。
- **🔒 高级加密识别**：能够自动识别并拦截开启了云服务“高级数据保护” (ARES 加密) 的笔记以防止生成出错误乱码数据，并给以人性化的明文指路提示。

## 📦 安装与前置依赖

### 前置要求 (Prerequisites)
- **Obsidian 桌面端**： `v1.5.0` 或更高版本。
- **移动端支持**：由于移动端底层安全沙箱限制，部分同步功能受限。插件内置了「移动端禁用」开关，允许您在移动设备上自动挂起插件以节省资源。
- **网络环境**：能够正常访问 `i.mi.com` 的网络。

### 安装步骤
1. 进入你 Obsidian 知识库 (Vault) 的插件目录：`.obsidian/plugins/`。
2. 在该目录下新建文件夹，命名为 `obsidian-minote-sync`。
3. 把编译打包的所有文件（须含 `main.js`, `manifest.json` 等）放入该文件夹。
4. 打开 **Obsidian设置 -> 第三方插件**，关闭“安全模式”。
5. 在下方插件列表开启 **Mi Note Sync**。

## ⚙️ 使用说明

### 一键安全鉴权
1. 打开 Obsidian -> 设置 -> Mi Note Sync。
2. 点击 **[点击前往授权]**，在弹出的定制容器内无感扫码或密码查验。
3. 提示绿色成功后，窗口将自动销毁。你现在的设备已通过极其安全的底层沙盒拿到了永久通行证。

### 同步执行
- **增量常规模式**：点击最左侧 Ribbon 工具栏的 **🔄 图标**，亦可使用 `Ctrl+P` 执行 `Sync Mi Notes`。底部状态栏会提供无感式的轻打扰实时进度追踪；如遇极其罕见的故障，会在大盘完结后触发汇总异常面板。
- **启动时自动同步**：在设置 -> **启动时自动同步** 开启后，每次打开 Obsidian，插件会在工作区完全就绪后于后台静默执行一次增量拉取，无需手动触发。
- **强制全量覆写**：如果你刚在云端整理了诸多分类或本地不慎误删了部分笔记，在设置面板的底部能够找到 **[立即执行全量同步]**，它能无视既往记录重新拉取全部文件。
- **附件策略配置**：支持在设置里自主切换"直接下载多媒体本体"或"引入云端源图在线网链"。
- **MiNote2Daily (小米笔记转日记)**：在侧边栏点击 `list-plus` 图标，可将今日所有更新的小米笔记按照预设模板同步至当日日记中，支持自动随主同步流程触发。

### 📝 模板引擎与格式自定义

在设置的 **「输出格式与模板」** 分区，可以按喜好完全自定义每篇导出笔记的命名格式、顶部属性结构以及聚合/日记模板。

**💡 变量辅助助手**：所有模板输入框下方均配有可用变量标签栏，点击即可在光标处插入对应的变量占位符，无需手动记忆。

**可用变量（文件名与 YAML 模板通用）**：

| 变量 | 含义 |
|---|---|
| `{{id}}` | 小米笔记唯一 ID |
| `{{title}}` | 提取自正文的笔记标题 |
| `{{folder}}` | 所属文件夹分类名 |
| `{{createTime}}` | 创建时间（格式由「时间格式化」选项决定） |
| `{{modifyTime}}` | 最后修改时间 |
| `{{color}}` | 背景颜色标记 ID |

**配置示例**：
- 文件名模板：`{{folder}}-{{title}}`
- 时间格式化：`YYYY-MM-DD HH:mm`
- Frontmatter 模板：
  ```yaml
  mi-note-id: {{id}}
  aliases: [{{title}}]
  tags: [MiNote/{{folder}}]
  created: {{createTime}}
  ```

> ⚠️ 若模板中不含时间戳或唯一 ID，不同笔记可能产生同名冲突。插件会自动在重名文件后追加 `_1`, `_2` 序号来规避覆盖，但仍建议在命名中保留一个唯一字段。  
> ⚠️ 修改模板后，**已同步在本地的历史笔记不会自动改名**，只有再次同步到更新的云端笔记才会触发新格式落地。如需全量重命名，请执行「强制全量同步」。

## ⚠️ “高级数据保护” 局限说明

如果您的账号开启了小米澎湃 OS 云服务的 **「高级数据保护（端到端加密）」**，这些核心数据在触及任何第三方网络及硬件前都会紧闭在坚不可摧的密文外壳内。因为本同步组件并不拥有只驻留在您物理手机上的原生环境私钥证书，受限于安全天堑，**无法做任何形式的暴力重组解密**。

**解决方案**：建议在完成整个知识库“最初始的大搬家”同步期间，在原手机终端 **短暂将其关闭**，让所有明文随云顺理成章地推入您的电脑库中；后续您便可以全盘依托 Obsidian 丰厚的本地加密生态系统来永久保障隐私。

## ❓ 常见问题答疑 (FAQ)

**Q：为什么有的笔记同步下来是一堆乱码提示，甚至告诉我“被 ARES 加密”了？**  
**A：** 当您在小米开启了**「高级数据保护（端到端加密）」**后，离开您本人的认证手机以外的所有数据都会变为坚不可摧的密文。这是正常现象。请去手机设置中**短暂关闭云端加密**并给云端十分钟解密时间后再度使用插件同步即可。

**Q：我疯狂点全量同步，会被云端封号吗？**  
**A：** 插件只采用与平时在 `i.mi.com` 常规翻页看笔记一样的频次进行正规读取，且加入了 `isSyncing` 控制防止并发泛滥，非常贴合自然人的阅览习惯。但仍请遵循个人备份原则文明调用，不要刻意连续数小时强制发起千篇体量的全量同步。
